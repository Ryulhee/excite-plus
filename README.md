# EXCITE plus Vital DB Front-end Prototype

병원 내부망 NAS 배포 전 화면·입력 흐름을 검증하기 위한 정적 HTML/CSS/JavaScript 프로토타입입니다.

## 이번 버전 핵심

- 연구 태그를 `연구 태그 → 환자를 나눌 기준 → 선택할 수 있는 값` 트리로 편집
- 드래그앤드롭, 상하 이동, 들여쓰기/내어쓰기, 접기/펼치기, 조건부 질문 미리보기
- 조건부 태그 경로를 사람·ECMO episode·검사기록·CSV에 부여
- 환자번호 중복 안내 및 기존 환자 열기
- ECMO episode 탭을 통한 기존 입력 조회·수정
- Episode 기본정보, Pre-ECMO 확장 검사, 시간 이벤트별 Vent/Flow/Vital 입력
- 미생물 결과 중복 확인과 관리자 중복 저장 사유
- 환자–태그–ECMO episode–CSV 통합 3열 매칭 화면
- 연결된 CSV를 드래그하여 시간 순서 변경, 기준 일시·시점명·episode 매칭 저장
- 분석 미리보기 접기/펼치기
- 분석 결과 상세·수정·삭제·그래프 PNG 다운로드
- 파일 등록대장 아래 3개월 ECMO 기간 캘린더와 태그 색상·이니셜 표시
- `schema-proposal.sql`에 MariaDB용 제안 스키마 포함

## 현재 저장 방식

현재는 브라우저 `localStorage`를 사용하는 프론트엔드 프로토타입입니다. 운영 버전에서는 `assets/app.js`의 `readState/writeState/patchState`를 병원 내부 NAS의 Python API 호출로 교체해야 합니다.

## 주요 파일

- `patient-upload.html`: 환자 master, ECMO episode, 검사 입력
- `tag-manager.html`: 연구 태그 트리 편집
- `tag-assignment.html`: 조건부 태그 경로 부여
- `patient-matching.html`: 환자–CSV 통합 관리
- `vital-upload.html`: Raw CSV 업로드·자동 변환·QC
- `schema-proposal.sql`: MariaDB 제안 스키마

실제 환자정보를 이 정적 프로토타입 또는 브라우저 localStorage에 저장하지 마세요. 운영 전 병원 전산·보안·IRB 검토가 필요합니다.

## 2026-07-07 화면 및 ECMO 입력 보완

- 모든 업무 화면에서 좌측 작업공간 사이드바를 제거하고 상단 메뉴만 사용합니다.
- 화면 폭을 전체 사용하도록 조정하고 표 안의 긴 값은 줄바꿈해 페이지 가로 스크롤을 줄였습니다.
- 환자 master 목록 행 또는 `수정` 버튼을 누르면 기존 생년월일, 성별, 키·몸무게, 과거력, 알레르기, 투약 내용을 다시 불러와 수정할 수 있습니다.
- Pre-ECMO: PH, PCO2, PO2, HCO3, SaO2, Lactate, Creatinine, Hgb, Plt, INR, APPT, AST, ALT, Bilirubin, Albumin, CRP, ESR, proBNP, Myoglobin, CK-MB, Troponin I, Troponin T와 SBP/DBP/Mean BP/PR 및 Vent 정보를 저장합니다.
- Intra-ECMO: GCS 전 Flow, 4시간 Flow·ABGA·Vent/Vital, 24시간 Flow·RPM·ECMO FiO2·Sweep gas·ABGA·Vent/Vital을 저장합니다.
- 파일 등록대장 환자 검색은 이니셜, 전체 등록번호 또는 등록번호 끝자리로 찾을 수 있습니다. 화면에는 등록번호가 마스킹되어 표시됩니다.
- ECMO 주요 시간 이벤트와 CSV의 시점명 또는 기준 일시가 일치하면 이벤트 카드 아래에 업로드 QC 그래프가 표시됩니다. 새로 업로드하는 CSV는 그래프용 축약 preview 데이터를 함께 저장합니다.


## 2026-07-07 CSV 모니터 배정 자동 매칭

- Raw CSV 업로드 화면을 좌측 `업로드·변환 상태`, 우측 `신호 궤적 미리보기` 구조로 변경했습니다.
- `SICU12_260707`, `CCU03-A1_260702` 형식에서 병동, 모니터 번호, 기록일을 자동 추출합니다.
- 날짜·병동·모니터·환자를 미리 등록하는 `모니터–환자 배정 관리`를 추가했습니다.
- 같은 날짜와 모니터에서 생성된 CSV는 해당 환자에게 자동 연결됩니다.
- 모니터 배정 목록과 업로드 상태 목록은 드래그하여 표시 순서를 변경할 수 있습니다.
- 수동 환자 매칭은 모니터 자동 배정보다 우선하며 자동 처리로 덮어쓰지 않습니다.

### 미매칭 중복 CSV 삭제
- 환자–CSV 통합 관리의 `매칭되지 않은 CSV`에서 중복으로 확인된 카드에만 `중복 CSV 삭제` 버튼이 표시됩니다.
- 동일 파일명·크기 또는 동일 case ID·기록 구간·크기가 일치하는 경우에만 중복으로 판단합니다.
- 삭제 전 비교 기록과 중복 기준을 확인하며, 삭제 후 파일 등록 상태와 미매칭 목록에서 함께 제거되고 감사기록이 남습니다.


## 2026-07-07 시간대별 모니터 배정 및 달력 표시

- 로그인 화면과 프론트엔드 코드에 초기 관리자 비밀번호 원문을 표시하지 않고 해시 비교 방식으로 변경했습니다.
- Raw CSV 화면 첫 줄을 `Raw CSV 자동 변환`과 `모니터–환자 배정 관리`의 좌우 2열 구조로 재배치했습니다.
- 업로드·변환 상태는 아래쪽 좌측, 신호 궤적 미리보기는 아래쪽 우측에 배치했습니다.
- 모니터 배정은 날짜뿐 아니라 시작·종료 시간을 저장하며, 같은 날 오전·오후에 서로 다른 환자를 배정할 수 있습니다.
- `SICU12_260707_083000.csv`와 같이 파일명에 포함된 시각을 읽어 해당 시간대 환자에게 자동 매칭합니다.
- 파일명에 시각이 없고 같은 날짜·모니터 배정이 여러 개인 경우에는 임의 매칭하지 않습니다.
- 기본 병동/방은 SICU, CCU, 이동형이며 사용자가 새 병동/방을 추가할 수 있습니다.
- 파일 등록대장 달력에서 환자 ECMO 기간은 태그색 실선, 모니터 사용 기간은 병동별 줄무늬 선으로 구분해 표시합니다.

## 2026-07 미매칭 CSV 검색 및 삭제

- 환자–CSV 통합 관리의 미매칭 CSV 목록에서 파일명, case ID, 기록일, 모니터, 업로드일을 직접 입력해 검색할 수 있습니다.
- 중복 파일은 `중복 CSV 삭제`, 일반 미매칭 파일은 `CSV 삭제` 버튼으로 제거할 수 있습니다. 연결된 CSV는 미매칭 목록에서 삭제할 수 없습니다.
- 삭제 시 vitalFiles, 파일 태그/부여 정보가 함께 제거되고 감사기록이 남습니다.

## 2026-07-07: Vital DB 시계열 통계 확장

- 통계 분석 화면 확장
  - 한 CSV 내부 구간/이벤트 전후 분석
  - 한 환자의 여러 CSV 연결 추세와 파일별 slope/AUC/변동성
  - 여러 환자의 반복측정 분석과 patient random intercept/slope 구조
  - lactate, ECMO flow, vasopressor, temperature, CRRT 등 임상조건 및 time interaction
  - resampling, timestamp/event alignment, 결측/이상치/평활화 설정

현재 프로토타입은 Vital DB 분석 사양과 파일 연결 metadata를 localStorage에 저장합니다. 실제 반복측정 통계 계산은 NAS 내부 Python API에서 실행하도록 연결해야 합니다.

## 2026-07-07 통계 분석 단순화 및 그래프 편집

- 통계 분석 시작 화면을 Vital DB 시계열 연구 중심으로 단순화했습니다.
  - 한 CSV 내부 구간 분석
  - 한 환자의 여러 CSV 연결 추세
  - 연구 태그를 이용한 환자 그룹별 반복측정 비교
- 사용자는 통계기법 대신 `무엇을 알고 싶은지`를 선택하며, 반복측정 혼합모형·시간 정렬·결측 처리는 자동 기본값으로 설정됩니다.
- LMM, 상호작용, resampling 같은 전문 설정은 `고급 설정` 안에 접어 두었습니다.
- 분석 결과 화면에서 그래프 제목, 부제목, X/Y축 제목, 글꼴, 글자 크기, 선 굵기, 범례, 눈금선, 투명 배경과 계열별 색상을 수정할 수 있습니다.
- 그래프는 웹페이지 설명을 포함하지 않고 실제 SVG 그래프 영역만 다운로드합니다.
  - PNG 4배 고해상도
  - SVG 벡터 파일(PowerPoint 확대·편집용)
  - 그래프 원자료 CSV
  - 분석 설명과 그래프 값을 포함한 TXT
- 실제 CSV preview data 또는 Python 분석 결과가 없는 경우에는 빈 예시 그래프를 다운로드하지 않도록 차단합니다.

## 2026-07-08 환자 master Excel형 일괄 등록

- 환자 master 입력 첫 화면을 행·열 방식의 대량 입력표로 변경했습니다.
- 환자는 세로 행, 기본정보와 과거력·알레르기·투약 항목은 가로 열로 배치합니다.
- Excel에서 여러 행·열을 복사하여 표의 시작 셀에 붙여넣을 수 있습니다.
- 열 제목이 포함된 경우 등록번호, 환자명, 성별 등 제목을 자동 인식하며 열 순서가 달라도 연결합니다.
- 등록번호 기준으로 신규 환자와 기존 환자 수정을 구분하고, 기존 환자의 빈 셀은 원래 값을 유지합니다.
- 입력표 내부 등록번호 중복, 날짜 형식, 키·몸무게 숫자 오류를 행별로 표시합니다.
- 기존 환자 전체 불러오기, 선택행 삭제, 10행 추가, 열 제목 복사, 실제 XLSX 템플릿 다운로드를 지원합니다.
- 개별 환자의 세부 과거력 메모를 수정할 수 있도록 기존 1명 상세 입력 폼은 접이식으로 유지합니다.

## v13 · ECMO 차수별 Clinical Workbook
- Patient Master 한 행은 한 환자의 한 ECMO 차수를 의미합니다.
- 동일 환자의 추가 ECMO는 같은 등록번호와 이니셜을 반복하고 ECMO 차수만 증가시킵니다.
- 상단 시트: Patient Master / n차 Pre-ECMO / n차 Intra-ECMO / n차 Outcome / n차 Microbiology.
- 각 시트는 직접 입력과 Excel 다중 행·열 붙여넣기를 모두 지원합니다.
- 미생물 검체·균종·결과는 추천값 선택과 자유 텍스트 입력을 함께 지원합니다.
